**Платформа:** PortSwigger Web Security Academy

---

## TL;DR

Приложение уязвимо к SQL-инъекции в форме авторизации. Изменив значение параметра username, удалось закомментировать проверку пароля и войти в систему под учетной записью administrator.

---

## Описание уязвимости

Форма входа использует пользовательский ввод при формировании SQL-запроса без параметризации. Это позволяет изменить структуру запроса и повлиять на логику аутентификации.

В данной лабораторной SQL-инъекция используется не для извлечения данных, а для обхода проверки пароля. С помощью комментария SQL остаток запроса игнорируется, поэтому сервер выполняет проверку только имени пользователя.

---

## Разведка

Приложение содержит стандартную форму авторизации с двумя полями:

- `username`
- `password`

Предположительно сервер выполняет запрос следующего вида:

```sql
SELECT * FROM users
WHERE username = '<username>'
AND password = '<password>';
```

Если пользовательский ввод не экранируется, появляется возможность изменить выполняемый SQL-запрос.

---

## Эксплуатация

Для выполнения атаки не потребовалось использовать Burp Suite — достаточно ввести SQL-инъекцию непосредственно в поле **Username**.

В поле **Username** вводим:

```text
administrator'--
```

Поле **Password** можно оставить любым.

После подстановки пользовательского ввода запрос будет выглядеть примерно так:

```sql
SELECT * FROM users
WHERE username = 'administrator'--'
AND password = 'password';
```

Последовательность `--` начинает SQL-комментарий, поэтому условие проверки пароля игнорируется.

Фактически сервер выполняет запрос:

```sql
SELECT * FROM users
WHERE username = 'administrator';
```

В результате происходит успешная авторизация под пользователем `administrator` без знания его пароля.

## Итог

Удалось выполнить SQL-инъекцию в механизме авторизации и войти под учетной записью `administrator` без знания пароля. Причиной уязвимости стало использование пользовательского ввода при построении SQL-запроса без параметризации.

## Защита

```python
cursor.execute(
    "SELECT * FROM users WHERE username = :username AND password = :password",
    {
        "username": username,
        "password": password
    }
)
```
